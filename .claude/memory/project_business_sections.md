업무는 크게 3개 섹션으로 구분됨.

## 1. 계약업체
- 일반 분석 의뢰 (ad-hoc)
- ETA `분석의뢰및결과` 테이블에 저장 (기존 구현)

## 2. 폐수배출업소
- 여수: 여수 산단 입주 기업들 (LG화학, 여천NCC, 롯데케미칼 등)
- 율촌: 율촌산단 + 세풍산단 입주 기업들
- 항목: BOD, TOC, SS, T-N, T-P, Phenols, N-Hexane
- DB 테이블: `폐수배출업소` (업체 마스터), `폐수배출업소_분석결과` (결과, 신규 생성)

## 3. 사업소 (처리시설 운영)
- 중흥처리장 (공공폐수처리시설)
- 월내처리장
- 율촌처리장
- 고정 일정에 따라 매일 시료 채취 및 분석 (요일별 항목 스케줄 있음)
- DB 테이블: `처리시설_마스터` (시료포인트+항목), `처리시설_분석결과` (신규 생성 예정)

## 분석자 배정 원칙
- 분장 테이블에서 **항목이 분석자에게 고정 배정** (분석자마다 고유 항목)
- 시료는 3개 섹션에서 섞여 오지만, 분석자는 본인 항목만 측정
- 예: A = BOD 전담, B = TN/TP 전담, C = SS/MLSS 전담 등

## 외부 연동 대상
- **ERP-U**: 폐수배출업소 분석결과 전송 (기존 UiPath → ETA 직접 연동으로 대체)
- **Zero4**: 처리시설 분석결과 입력 웹플랫폼 → ETA에서 자동 전송
- **측정인**: ETA에 부분 구현됨

## 통합 목표
> ETA 한 곳에 결과 입력 → ERP-U / Zero4 자동 전송 + 시험성적서 자동 생성

**Why:** 섹션별 분산 워크플로우(개인 엑셀, ERP 수동입력, Zero4 수동입력)를 ETA 단일 플랫폼으로 통합
**How to apply:** 분석자별 의뢰 목록 = 분장 테이블 기준으로 3개 섹션 시료를 통합 생성
