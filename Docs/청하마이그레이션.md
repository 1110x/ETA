### [2] 의뢰번호 시트 → 비용부담금_결과 INSERT

#### 엑셀 시트 구조
```
A열: 채수일자 (Excel 날짜 시리얼)
B열: 업체명(시료명), C열: SN(의뢰번호)
D열: 업체명(시료명), E열: SN(의뢰번호)
F열: 업체명(시료명), G열: SN(의뢰번호)
H열: 업체명(시료명), I열: SN(의뢰번호)
... (2컬럼 간격으로 반복, SN이 비어있으면 해당 행 종료)
```

#### DB 매핑 (비용부담금_결과)
| 엑셀 | DB 컬럼 | 비고 |
|------|---------|------|
| A열 날짜 시리얼 | 채수일 | yyyy-MM-dd 변환 |
| C/E/G/I... | SN | 의뢰번호 |
| B/D/F/H... | 업체명 | 시료명 |
| SN 접두사 판별 | 구분 | [율촌]→율촌, [세풍]→세풍, 없으면→여수 |
| MAX(순서)+1 | 순서 | 채수일+구분 기준 자동증가 |

#### SN 체계
- 여수: `MM-DD-NN` (예: 01-09-01)
- 율촌: `[율촌]MM-DD-NN`
- 세풍: `[세풍]MM-DD-NN`

#### 구현: `LoadRequestNumberData()`

### [3] BOD-DATA → 생물학적_산소요구량_시험기록부 + 비용부담금_결과.BOD

#### 엑셀 시트 구조 (실제 확인)
```
A열 (col 1): 채수일 ("2020. 1. 1. 오전 12:00:00" 형식)
B열 (col 2): BOD-분석자
C열 (col 3): 담당자-2
D열 (col 4): 시료량
E열 (col 5): 15min DO
F열 (col 6): 5Day DO
G열 (col 7): 식종액 BOD
H열 (col 8): 식종액함유량
I열 (col 9): 희석수 시료량
J열 (col 10): 식종수 D1
K열 (col 11): 식종수 D2
col 12~29: 기타 참고값
```

#### 샘플 블록 구조 (col 30부터, 8컬럼 간격)
| 오프셋 | 실제컬럼(블록0) | 내용 |
|--------|--------------|------|
| +0 | 30 (AD) | 시료명 (업체명) |
| +1 | 31 (AE) | 시료량(V) |
| +2 | 32 (AF) | D1 |
| +3 | 33 (AG) | D2 |
| +4 | 34 (AH) | F(x/y) |
| +5 | 35 (AI) | P |
| +6 | 36 (AJ) | **Result (결과값)** |
| +7 | 37 (AK) | Remark = **SN** (기존 엑셀에서 비고란에 SN 기입) |

블록1: col 30~37, 블록2: col 38~45, 블록3: col 46~53 ...

#### 처리 로직
1. A열 날짜 없으면 해당 행 종료
2. 날짜 변환 실패 시 skip (계속)
3. col 30 (시료명) 없으면 해당 행 블록 전체 종료
4. col 36 (결과) 없거나 "0"이면 skip
5. col 37 (SN) 접두사로 구분 판별
6. `UpsertBodData()` → `생물학적_산소요구량_시험기록부` 저장
7. `UpdateWasteResult()` → `비용부담금_결과.BOD` UPDATE (채수일+SN 매칭)

#### 생물학적_산소요구량_시험기록부 저장 컬럼
```
분석일, SN, 업체명, 구분, 소스구분, 시료명,
시료량, D1, D2, 희석배수(P), 결과,
식종시료량, 식종D1, 식종D2, 식종BOD, 식종함유량,
비고, 등록일시
```

#### 구현: `LoadBodData()`

## 날짜 변환

엑셀 날짜가 두 가지 형식으로 저장됨:

1. **Excel 날짜 시리얼** (숫자): `43831` → `2020-01-01`
   - 1900-02-29 버그: 시리얼 > 59이면 -1 보정
   - baseDate = 1899-12-30

2. **문자열 형식**: `"2020. 1. 1. 오전 12:00:00"`
   - ". 오" 패턴으로 시간 부분 제거
   - "yyyy. M. d" 파싱

구현: `TryParseExcelDate()`

## BOD-DATA → 비용부담금_결과 BOD UPDATE 로직

### 핵심 문제
- SN 형식 `MM-DD-NN`에서 MM=채수월, DD=채수일 → 채수일과 분석일이 다를 수 있음
- 예) SN `09-04-01` = 9월 4일 채수, 분석일은 9월 30일
- SN+업체명만으로 조회하면 다른 연도 데이터와 충돌 가능

### 해결: 분석일 기준 날짜 범위 조회

`UpdateWasteResult(분석일, SN, ...)`:
- 채수일 검색 범위: `분석일 - 40일 ~ 분석일` BETWEEN 조건
- SN 매칭
- 12월 채수 → 다음 해 1월 분석일도 자동 커버 (날짜 범위 조회로 해결)

```sql
SELECT Id FROM `비용부담금_결과`
WHERE SN=@sn
  AND 채수일 BETWEEN @start AND @end   -- 분석일-40일 ~ 분석일
LIMIT 1
```

## 주의사항

1. **SN 없는 행**: 결과값은 있지만 SN(Remark)이 없는 경우 → `비용부담금_결과` UPDATE 스킵, 시험기록부에만 저장
2. **결과값 0**: skip 처리
3. **중복 방지**: 시험기록부는 `분석일+SN+시료명` 기준 UPSERT (재실행 시 UPDATE)
4. **구분 판별**: SN 접두사 기준 (없으면 여수)
5. **채수일 ≠ 분석일**: `UpdateWasteResult`에 분석일을 넘겨야 함, 채수일 직접 사용 금지
