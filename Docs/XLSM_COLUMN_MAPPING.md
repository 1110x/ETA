# xlsm 컬럼 매핑 - 항목별 상세 분석

## 요약

xlsm 파일은 **2페이지 구조**로 설계되어 있습니다:
- **페이지 1**: 참고용 데이터 (표준물질, 계산식, 헤더 등)
- **페이지 2**: 실제 분석 샘플 데이터 (반복되는 블록 구조)

각 시트는 **~10 컬럼 단위로 반복되는 샘플 블록**을 포함하고 있습니다.

| C | 행1 | `담당자-2` | - |

| D | 행2+ | 숫자 | BOD 시험용 시료량 |

| E-O | 행2+ | 숫자 | 표준물질, DO(15분), DO(5일), 희석배수, 기울기, 절편, R² 등 |

**블록 구조**: `시료명 → 시료량(V) → D1 → D2 → F(x/y) → P → Result → Remark`

| 블록# | 시료명 컬럼 | 시료량 | D1 | D2 | F(x/y) | P | 결과 | 비고 |
|-------|----------|--------|-----|-----|---------|---|------|------|
| 1 | P (16) | Q (17) | R (18) | S (19) | T (20) | U (21) | **V (22)** | W (23) |
| 2 | X (24) | Y (25) | Z (26) | BA (27) | BB (28) | BC (29) | **BD (30)** | BE (31) |
| 3 | BF (32) | BG (33) | BH (34) | BI (35) | BJ (36) | BK (37) | **BL (38)** | BM (39) |
| ... | +8 | +8 | +8 | +8 | +8 | +8 | **+8** | +8 |

**DB 저장**:
- 열 A (행2+): 채수일
- **열 V, BD, BL, ... (8컬럼 단위 +6)**: `BOD` 결과값

**예시 (행 2)**:
```
A2: 43831 (2020-01-01)

P2: 104.9 (시료명? - 실제는 숫자)
Q2: 37104 (시료량)
...
V2: 4.08 (BOD 결과)  ← DB 저장 대상
```

## 반복 블록 계산 (프로그래밍)

### BOD-DATA (8컬럼 블록)
```
블록 N의 시작 컬럼 = 16 + (N-1)*8 = P + (N-1)*8
결과 컬럼 = 시작 컬럼 + 6
```
- 블록1: P16 (결과: V22)
- 블록2: X24 (결과: BD30)
- 블록3: BF32 (결과: BL38)

### SS-DATA, TN-DATA, TP-DATA, N-Hexan-DATA, TOC, Phenols (7컬럼 블록)
```
블록 N의 시작 컬럼 = 시작위치 + (N-1)*7
결과 컬럼 = 시작 컬럼 + 5 (또는 +6)
```
- 블록1: C3 (결과: H8)
- 블록2: J10 (결과: O15)
- 블록3: Q17 (결과: V22)

## 주의 사항

1. **Excel 날짜 시리얼 변환**
   - 엑셀: 1 = 1900-01-01
   - 1900-02-29 버그: 시리얼 > 59면 -1 일 조정
   - 예: 43831 = 2020-01-01

2. **시료명 추출**
   - xlsm 시료명이 정확하지 않을 수 있음
   - 실제 데이터: 숫자 ID (예: 104.9) 또는 불완전한 텍스트
   - DB 매칭: 채수일 + 시료명 + 시료량으로 중복 제거

3. **특수 로직**
   - **TOC**: TCIC 우선, IC > TC×50% → NPOC 대체
   - **Phenols**: 결과값 기준으로 직접법/추출법 선택

4. **빈 행 종료**
   - 채수일(열 A)이 비어있으면 데이터 종료
   - 각 시트별 실제 행 수: BOD(2804), SS(2352), TN(2364), TP(2875), N-Hexan(2296), TOC(2558), Phenols(2573)

5. **시료명 매칭 문제**
   - xlsm 업체명 → DB 폐수배출업소 자동 매칭 불확실
   - 수동 매칭 또는 SN으로만 저장하는 방식 권장
