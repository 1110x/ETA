# 분석결과입력 시스템 설계

## 개요
분석인원이 장비별로 엑셀 결과파일을 첨부하면, 의뢰 목록(폐수의뢰및결과)과 자동 매칭/검증 후 일괄 입력하는 시스템.

## 메뉴 구조

### 메인메뉴 > 분석결과입력
13개 하위 메뉴 (장비/분석법 기준):

| Key | 메뉴명 | 매핑 항목 | 비고 |
|-----|--------|----------|------|
| BOD | BOD | BOD | ES 04305.1c |
| TOC | TOC | TOC | TC-IC / NPOC |
| SS | SS | SS | |
| UVVIS | UV VIS | T-N, T-P, Phenols | 자외선-가시광선 분광광도계 |
| NHEX | NHexan | N-Hexan | 노말헥산추출물 |
| GCMS | GCMS | (확장 예정) | 기체크로마토그래프-질량분석기 |
| ICP | ICP | (확장 예정) | 유도결합플라즈마 |
| IC | IC | (확장 예정) | 이온크로마토그래프 |
| LCMS | LCMS | (확장 예정) | 액체크로마토그래프-질량분석기 |
| CN | CN | (확장 예정) | 시안 |
| AAS | AAS | (확장 예정) | 원자흡수분광광도계 |
| COLOR | COLOR | (확장 예정) | 색도 |
| ECO | 생태독성 | (확장 예정) | |

### 서브메뉴 버튼
- **새로고침** (BT1): 트리뷰 + 그리드 재로드
- **검증** (BT2): 현재 카테고리 파일 재첨부 + 검증
- **입력** (BT3): 검증된 데이터 일괄 DB 반영 (토글 ON만)
- **출력** (BT4): 현재 그리드 클립보드 복사

## 4-패널 레이아웃

### Show1 (좌측 트리)
- 상단: 카테고리 버튼 WrapPanel (13개)
  - 클릭 → 파일 피커 열림 → 엑셀 파싱 → Show2 로드
  - 파일 첨부 완료: ✅ 아이콘 + 녹색 배경
  - 미첨부: 기본 아이콘
- 하단: 날짜 트리뷰 (월별 그룹, 입력현황 배지)

### Show2 (중앙 그리드)

#### 파일 미첨부 시
일반 그리드: 구분 | S/N | 업체명 | (활성 항목 컬럼들)

#### 파일 첨부 시 (검증 그리드)
| 검증 | 토글 | SN | 시료명 | 결과값 | 의뢰SN |
|------|------|-----|-------|--------|--------|

- **검증 아이콘 (왼쪽)**:
  - 🟢 의뢰 존재 + 해당 항목 미입력 → 입력 가능
  - 🟡 의뢰 존재 + 이미 결과 있음 → 덮어쓰기
  - 🔴 의뢰에 해당 SN 없음 → 미매칭

- **토글스위치**: 개별 시료 입력/제외 결정
  - ON (기본값): 입력 대상
  - OFF: 입력에서 제외
  - 미매칭(🔴) 행은 자동 OFF + 비활성화
  - 테마색 적용 (BtnPrimaryBg)

### Show3 (우측 편집)
선택한 시료의 항목별 입력 폼
- 정보 배지: S/N, 구분, 채수일, 관리번호
- 항목별 TextBox (Enter→다음 필드, 마지막 Enter→저장)
- 검증 그리드 클릭 시 결과값 자동 채움

### Show4 (하단 통계)
- 항목별 입력 프로그레스 바
- 📎 표시: 해당 항목의 카테고리에 파일 첨부됨
- 구분별(여수/율촌/세풍) 요약

## 엑셀 파일 구조 (BOD.xlsx 기준)

### 2-page 구조
```
페이지1 (A~H열):
  행1: 문서번호, 날짜, ...
  행2: 분석방법
  행5: 헤더 (시료명, 시료량, D1, D2, f(x/y), P, Result, Remark)
  행6: 식종수의 BOD (제외)
  행7: SCF (제외)
  행8~: 데이터
    A열 = 시료명 (업체명)
    G열 = Result (분석 결과값, mg/L)
    H열 = Remark (SN: "02-10-01", "[세풍]02-10-01")

페이지2 (I~P열): 동일 구조
  I열 = 시료명
  O열 = Result
  P열 = Remark (SN)
```

### SN 매칭 규칙
- 엑셀 H/P열(Remark) → `폐수의뢰및결과.SN`과 1:1 매칭
- SN 형식: `MM-DD-순서` (여수), `[세풍]MM-DD-순서`, `[율촌]MM-DD-순서`
- SN 미매칭 시 업체명으로 재시도

### 필터링 규칙
- "식종", "SCF", "분석담당" 포함 행 제외
- 빈 행(시료명+SN 모두 공백) 도달 시 중단
- Result가 숫자면 소수점 1자리로 포맷

## DB 구조

### 폐수의뢰및결과 테이블
```sql
Id, 채수일, 구분, 순서, SN, 업체명, 관리번호,
BOD, TOC, SS, T-N, T-P, N-Hexan, Phenols, 비고, 확인자
```

### 서비스 메서드
- `WasteSampleService.GetDates()` — 날짜 목록
- `WasteSampleService.GetByDate(채수일)` — 날짜별 전체 행
- `WasteSampleService.UpdateValues(id, bod, toc, ss, tn, tp, nHexan, phenols)` — 결과값 업데이트

## 신규 항목 확장 가이드

새 분석항목(GCMS, ICP 등) 추가 시:

1. **Categories 배열**에 항목 추가 (WasteAnalysisInputPage.axaml.cs)
   ```csharp
   ("GCMS", "GCMS", "📊", new[] { "Benzene", "Toluene", ... }),
   ```

2. **WasteSample 모델**에 속성 추가 (Models/WasteSample.cs)

3. **DB 테이블**에 컬럼 추가 (폐수의뢰및결과)

4. **GetSampleValue() switch문** 확장

5. **UpdateValues() 파라미터** 확장 또는 별도 메서드 추가

6. **엑셀 파싱 구조**가 다른 장비면 `ParsePage()` 오버로드 추가
   - 각 장비별 엑셀 포맷이 다를 수 있음 (컬럼 위치, 시작행 등)
   - Categories 정의에 파서 타입 필드 추가 검토

## 흐름 요약
```
카테고리 버튼 클릭
  → 파일 피커 열림
  → 엑셀 파싱 (ParsePage)
  → 카테고리별 데이터 저장
  → 날짜 선택 시 SN 기반 의뢰 매칭
  → Show2 검증 그리드 표시 (🟢🟡🔴 + 토글)
  → "입력" 버튼 → 토글 ON인 행만 DB 반영
```
