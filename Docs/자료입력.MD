# 분석결과입력 시스템 설계

## 개요

### 서브메뉴 버튼
- **새로고침** (BT1): 트리뷰 + 그리드 재로드
- **검증** (BT2): 현재 카테고리 파일 재첨부 + 검증
- **입력** (BT3): 검증된 데이터 일괄 DB 반영 (토글 ON만)
- **출력** (BT4): 현재 그리드 클립보드 복사

## 4-패널 레이아웃

### Show1 (좌측 트리)
- 상단: 카테고리 버튼 WrapPanel (13개)
  - 클릭 → 파일 피커 열림 → 엑셀 파싱 → Show2 로드
  - 파일 첨부 완료: ✅ 아이콘 + 녹색 배경
  - 미첨부: 기본 아이콘
- 하단: 날짜 트리뷰 (월별 그룹, 입력현황 배지)

### Show2 (중앙 그리드)

#### 파일 미첨부 시
일반 그리드: 구분 | S/N | 업체명 | (활성 항목 컬럼들)

#### 파일 첨부 시 (검증 그리드)
| 검증 | 토글 | SN | 시료명 | 결과값 | 의뢰SN |
|------|------|-----|-------|--------|--------|

- **검증 아이콘 (왼쪽)**:
  - 🟢 의뢰 존재 + 해당 항목 미입력 → 입력 가능
  - 🟡 의뢰 존재 + 이미 결과 있음 → 덮어쓰기
  - 🔴 의뢰에 해당 SN 없음 → 미매칭

- **토글스위치**: 개별 시료 입력/제외 결정
  - ON (기본값): 입력 대상
  - OFF: 입력에서 제외
  - 미매칭(🔴) 행은 자동 OFF + 비활성화
  - 테마색 적용 (BtnPrimaryBg)

### Show3 (우측 편집)
선택한 시료의 항목별 입력 폼
- 정보 배지: S/N, 구분, 채수일, 관리번호
- 항목별 TextBox (Enter→다음 필드, 마지막 Enter→저장)
- 검증 그리드 클릭 시 결과값 자동 채움

### Show4 (하단 통계)
- 항목별 입력 프로그레스 바
- 📎 표시: 해당 항목의 카테고리에 파일 첨부됨
- 구분별(여수/율촌/세풍) 요약

## 엑셀 파일 구조 (BOD.xlsx 기준)

### 2-page 구조
```
페이지1 (A~H열):
  행1: 문서번호, 날짜, ...
  행2: 분석방법
  행5: 헤더 (시료명, 시료량, D1, D2, f(x/y), P, Result, Remark)
  행6: 식종수의 BOD (제외)
  행7: SCF (제외)
  행8~: 데이터
    A열 = 시료명 (업체명)
    G열 = Result (분석 결과값, mg/L)
    H열 = Remark (SN: "02-10-01", "[세풍]02-10-01")

페이지2 (I~P열): 동일 구조
  I열 = 시료명
  O열 = Result
  P열 = Remark (SN)
```

### SN 매칭 규칙
- 엑셀 H/P열(Remark) → `폐수의뢰및결과.SN`과 1:1 매칭
- SN 형식: `MM-DD-순서` (여수), `[세풍]MM-DD-순서`, `[율촌]MM-DD-순서`
- SN 미매칭 시 업체명으로 재시도

### 필터링 규칙
- "식종", "SCF", "분석담당" 포함 행 제외
- 빈 행(시료명+SN 모두 공백) 도달 시 중단
- Result가 숫자면 소수점 1자리로 포맷

## 생태독성 (ECO) — TSK/Probit 통계분석

### 개요
물벼룩 급성독성시험(ES 04704.1c) 데이터에서 LC50을 계산하고 TU(=100/LC50)로 변환하여 저장.
두 가지 통계 방법 지원: **TSK (Trimmed Spearman-Karber)**, **Probit**.

### 데이터 입력 방식
1. **직접 입력** — Show3에서 농도/생물수/사망수를 입력하고 "TSK 계산" 또는 "Probit 계산" 버튼 클릭
2. **엑셀 파일** — ECO 카테고리 파일 첨부 (추후 파서 구현)

### Show3 직접 입력 UI
```
시험종: [물벼룩]    시험시간: [24] H
대조군: 생물수 [20]  사망수 [0]
───────────────────────────────────
#   농도(%)     생물수    사망수
1   [6.25]      [20]      [0]
2   [12.5]      [20]      [1]
3   [25.0]      [20]      [0]
4   [50.0]      [20]      [0]
5   [100.0]     [20]      [16]
───────────────────────────────────
[TSK 계산]  [Probit 계산]

TU = 1.3  (LC50 = 76.69)
95% CI: 69.42 ~ 84.71  |  TSK  |  Trim: 20%
```

### 통계 알고리즘

#### TSK (Trimmed Spearman-Karber)
1. 반응비율 계산: `p_i = 사망수_i / 생물수_i`
2. Abbott 보정: `p = (p - Pc) / (1 - Pc)` (Pc = 대조군 사망률)
3. 단조증가 보정 (PAVA isotonic regression)
4. 자동 trim % 결정
5. Spearman-Karber 추정: `m = Σ[(logC_{i+1} - logC_i) × p_mid]`
6. `LC50 = 10^(logC_max - m)`
7. 분산: `Var = d² × Σ[p(1-p)/(n-1)]`
8. 95% CI: `10^(logLC50 ± 1.96 × SE)`

#### Probit
1. 반응비율 + Abbott 보정 + Miller-Tainter 에지케이스 처리
2. Probit 변환: `Y = Φ⁻¹(p) + 5`
3. 가중최소제곱 반복회귀: `Y = a + b × log₁₀(X)`
4. `LC50 = 10^((5-a)/b)`
5. 95% CI: 분산-공분산 행렬 기반

### 결과값
- **TU = 100 / LC50** (소수 1자리)
- DB 저장: `생태독성_시험기록부` 테이블
- 처리시설/폐수 결과: `TU` 값으로 저장

### DB 테이블 (`생태독성_시험기록부`)
```sql
id, 분석일, SN, 업체명, 시료명, 구분, 소스구분,
시험종, 시험시간, 시험시간단위,
대조군_생물수, 대조군_사망수,
농도_1~8, 생물수_1~8, 사망수_1~8,
LC50, LC50_하한, LC50_상한, TU,
분석방법, trim_percent, 결과,
비고, 등록일시
```
