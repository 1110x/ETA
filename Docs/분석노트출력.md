# 분석노트 출력 설계

## 개요

처리시설은 **매주 동일한 루틴**으로 분석이 진행되므로,  
분석의뢰서(분석노트) 출력 시 해당 채수일의 요일을 기반으로  
**처리시설 분석항목 + 담당자**를 자동 포함하여 출력한다.

---

## 출력 대상 데이터

### 1. 비용부담금 채수 의뢰 (WasteSample)
| 필드 | 소스 | 비고 |
|------|------|------|
| 채수일 | `WasteSample.채수일` | yyyy-MM-dd |
| 구분 | `WasteSample.구분` | 여수 / 율촌 / 세풍 |
| SN | `WasteSample.SN` | 03-31-01 / [세풍]03-31-01 |
| 업체명 | `WasteSample.업체명` | |
| 관리번호 | `WasteSample.관리번호` | |
| 확인자 | `WasteSample.확인자` | |

### 2. 처리시설 분석항목 루틴 (AnalysisPlan — 분석계획)
| 필드 | 소스 | 비고 |
|------|------|------|
| 분석항목 | `_analysisPlanState` 키 | 예) BOD, TOC, T-N ... |
| 요일 활성 여부 | `_analysisPlanState[항목][0..6]` | 0=월 … 4=금 |

> `_analysisPlanDays = { "월","화","수","목","금","토","일" }`

### 3. 처리시설 결과행 (FacilityResultRow)
| 필드 | 소스 | 비고 |
|------|------|------|
| 시료명 | `FacilityResultRow.시료명` | 처리시설명 |
| 비고마스터 | `FacilityResultRow.비고마스터` | "주1회 목요일" 등 주기 메모 |
| 항목별 활성 | `BOD활성`, `TOC활성`, ... | 마스터에서 "O" 설정된 항목만 True |

### 4. 분석 담당자 (AnalysisRequestService)
| 메서드 | 반환 | 비고 |
|--------|------|------|
| `GetManagersByDate(dateKey)` | `Dictionary<string, string>` | 분석항목 전체명 → 담당자 이름 |

---

## 출력 로직 흐름

```
채수일(yyyy-MM-dd) 선택
    │
    ├─ 1. 요일 계산
    │      DayOfWeek dow = DateTime.Parse(채수일).DayOfWeek
    │      int dayIdx = (int)dow == 0 ? 6 : (int)dow - 1   // 0=월 … 6=일
    │
    ├─ 2. 비용부담금 채수 목록 로드
    │      WasteSampleService.GetByDate(채수일)
    │      → 구분(여수/율촌/세풍)별 분류
    │
    ├─ 3. 해당 요일에 체크된 분석항목 조회
    │      _analysisPlanState.Where(kv => kv.Value[dayIdx] == true)
    │                         .Select(kv => kv.Key)
    │      → 분석항목 목록(예: BOD, TOC, SS, T-N, T-P)
    │
    ├─ 4. 담당자 조회
    │      AnalysisRequestService.GetManagersByDate(채수일)
    │      → 분석항목 → 담당자 매핑
    │
    └─ 5. Excel 출력 (ClosedXML)
           PrintAnalysisNote(채수일, samples, activeItems, managers)
```

## 구현 계획

### Phase 1 — 분석계획 DB 저장 (현재: 메모리만)

- **테이블**: `처리시설_분석계획`
  ```sql
  CREATE TABLE 처리시설_분석계획 (
      Id        INT AUTO_INCREMENT PRIMARY KEY,
      분석항목  VARCHAR(100) NOT NULL,
      요일      TINYINT NOT NULL,   -- 0=월 … 6=일
      활성      TINYINT(1) DEFAULT 1,
      수정일    DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
  );
  ```
- **서비스**: `FacilityAnalysisPlanService`
  - `Save(Dictionary<string, bool[]> state)`
  - `Load() → Dictionary<string, bool[]>`

### Phase 2 — 출력 서비스 구현

- **파일**: `Services/SERVICE2/AnalysisNoteExportService.cs`
- **메서드**:
  ```csharp
  public static void Export(
      string 채수일,
      List<WasteSample> samples,
      Dictionary<string, bool[]> planState,
      Dictionary<string, string> managers,
      string outputPath)
  ```
- **의존**: `ClosedXML` (기존 사용 중)

### Phase 3 — UI 연결

- `WasteSampleListPage`의 서브메뉴 또는 상세 패널에 **"분석노트 출력"** 버튼 추가
- 클릭 시 `FileSavePicker` → `AnalysisNoteExportService.Export()` 호출

## 미결 사항 (확인 필요)

1. **분석계획 요일별 항목** — 현재 `_analysisPlanState`는 앱 재실행 시 초기화됨  
   → Phase 1 완료 전까지 Excel(`요일별 분석항목 정리.xlsx`) 읽어서 초기 상태 구성 가능

2. **처리시설 ↔ 분석항목 매핑** — `FacilityResultRow`의 `항목활성` 필드로 시설별 활성 항목을 구분  
   → 출력 시 "시료명(처리시설)" 기준으로 활성 항목만 행에 표시

3. **담당자 매핑 정확도** — `GetManagersByDate`는 분석의뢰 DB 기반  
   → 처리시설 담당자와 수질분석센터 담당자가 다를 경우 별도 컬럼 필요 여부 검토
