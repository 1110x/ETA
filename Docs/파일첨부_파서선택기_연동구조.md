# 파일첨부 = 파서선택기 연동 구조

## 핵심 개념

분석결과 입력(WasteAnalysisInputPage)에서 파일을 첨부하면
시그니처 DB를 먼저 뒤지고, 없으면 ONNX 모델을 보조로 쓴다.

```
[파일 첨부]
    ↓ 텍스트 추출 (CSV/TXT/XLSX/PDF)
    ↓
    ├─ SignatureClassifier.ClassifyTopK(text, 3)   ← 1순위: DB 규칙
    └─ AiParserClassifier.PredictTopK(text, 3)    ← 2순위: ONNX (보완)
    ↓
    통합 후보 목록 다이얼로그 표시 (배지: 시그니처 | AI)
    ↓
    사용자 선택 → _classifierToRunner[key] → 파서 실행
```

---

## 시그니처란

파서 생성기에서 파일을 분석하면 추출되는 **키워드 집합**이다.

| 키워드 종류 | 예시 | 가중치 |
|------------|------|--------|
| 헤더 키워드 | "Absorbance", "Dilution", "Result" | ×3 (가장 중요) |
| 표준물질 키워드 | "STD", "Blank", "Cal", "검량" | ×2 |
| 시료명 키워드 | 시료 행의 첫 열 패턴 | ×1 |

**점수 계산:**
```
score = (매칭된 키워드 가중치 합) / (전체 가중치 합)
임계값: score ≥ 0.15 → 분류 성공
```

## 시그니처가 DB에 들어가는 경로 2가지

### 경로 1: 파서 생성기 → 와이어 연결 → 시그니처 저장

```
파서 생성기 탭
  └─ 파일 업로드 → 열 자동 감지
  └─ 와이어 연결 UI: 감지된 열 ──선── ETA 표준 필드 (SN, 시료명, 흡광도...)
  └─ Show3 패널 [시그니처 저장] 버튼
        ↓
        ParserSignature {
            ParserKey      = "BOD" (사용자가 ComboBox에서 선택)
            HeaderKeywords = 매핑된 열 헤더들  ← 와이어 연결에서 추출
            StdKeywords    = 검량선 행 키워드  ← 자동 감지된 CAL/STD/Blank 등
            Delimiter      = 감지된 구분자
            HeaderLines    = 감지된 헤더 행 수
        }
        ↓
        SignatureClassifier.SaveSignature(sig)  → DB: 파서_시그니처 테이블
```

### 경로 2: AI 문서분류 학습 완료 후 자동 빌드

```
AI 문서분류 탭
  └─ 학습 데이터 추가 (파서 타입별 샘플 파일)
  └─ [학습] 버튼 클릭
        ↓
        Python: TF-IDF + LogisticRegression → 분석항목_분류.onnx
        ↓ (학습 성공 시 자동)
        SignatureClassifier.BuildFromTrainingData()
          - AI_문서분류_학습데이터 테이블에서 파서별 텍스트 로드
          - 헤더 키워드: 처음 20행에 반복 등장하는 단어
          - 표준물질 키워드: STD/Cal/Blank/BLK 등 접두사 검출
          ↓
          SaveSignature() → DB 저장
```

## 분류 흐름 상세 (WasteAnalysisInputPage)

```csharp
// AttachExcel() 진입 조건
if (!SignatureClassifier.HasSignatures() && !AiParserClassifier.IsModelReady())
    return;  // 시그니처도 ONNX도 없으면 파서 자동 분류 건너뜀

// AiPickAndPredictAsync() 내부
var hits = new List<(string Label, float Prob, string Source)>();

// 1순위: 시그니처 분류기
if (SignatureClassifier.HasSignatures())
    foreach (var h in SignatureClassifier.ClassifyTopK(text, 3))
        hits.Add((h.ParserKey, h.Score, "시그니처"));

// 2순위: ONNX (중복 키 제외)
if (AiParserClassifier.IsModelReady())
    foreach (var (label, prob) in AiParserClassifier.PredictTopK(text, 3))
        if (!existKeys.Contains(label))
            hits.Add((label, prob, "AI"));

// 후보 다이얼로그 (배지 색: 시그니처=초록, AI=파랑)
→ 사용자 선택
→ _classifierToRunner["BOD"] = "Excel"  → 엑셀 파서 실행
→ _classifierToRunner["TOC_Shimadzu"] = "TOC_Shimadzu"  → TOC 파서 실행
```

## 파서 키 → 실행 파서 매핑 (_classifierToRunner)

| 파서 키 | 실행 파서 | 비고 |
|---------|----------|------|
| BOD, SS, NHex, UVVIS | Excel | 엑셀 범용 파서 |
| TOC_Shimadzu, TOC_Shimadzu_PDF | TOC_Shimadzu | 시마즈 TOC |

| TOC_Scalar_NPOC | Scalar_NPOC | 스칼라 NPOC |
| TOC_Scalar_TCIC | Scalar_TCIC | 스칼라 TCIC |
| GC | GC | GC 크로마토그래피 |
| UV_Shimadzu_PDF, UV_Shimadzu_ASCII | Shimadzu_UV | 시마즈 UV |
| UV_Cary_PDF, UV_Cary_CSV | Agilent_Cary | 애질런트 Cary |
| ICP | ICP_PDF | ICP |
| LCMS | LCMS_PFAS | LCMS |

## 신규 장비 파서 등록 워크플로

```
1. 장비에서 결과 파일 출력 (CSV/TXT/XLSX/PDF)
2. ETA → 파서 생성기 탭 → 파일 업로드
3. 자동 분석: 구분자 / 헤더 행 수 / 열 목록 감지
4. 와이어 연결 UI:
     감지된 열들 (좌)  ──●───●──  ETA 표준 필드 (우)
     "Absorbance"    ────────→  흡광도
     "SampleID"      ────────→  시료명
     "DilFactor"     ────────→  희석배수
5. Show3: 파서 타입 선택 (ComboBox) → [시그니처 저장]
6. 이후 동일 형식 파일 첨부 시 "시그니처" 배지로 자동 분류됨
```
