```

    ↓
DB 초기화 단계
    ├─ [Step 1-4] 기존 초기화 작업...
    │
    ├─ [Step 5] Phase 1 마이그레이션 ← NEW
    │   ├─ DbPhase1Migration.ExecutePhase1()

    │   ├─ RENAME TABLE (마이그레이션 팩토리 패턴)

    ↓
```

## 데이터 흐름 변화

### 분석결과 입력 (분석결과입력 페이지)

**Before**:
```
파일 업로드 → 파서 (UvvisXlsxParser, IcpCsvParser 등)
    ↓
[어느 테이블인가?] 불명확한 매핑
    ↓
분석의뢰및결과 OR 폐수의뢰및결과 OR 처리시설_측정결과
(테이블명이 혼재되어 있음)
    ↓
시험기록부 저장 (공용 *_시험기록부 테이블)
```

**After**:
```
파일 업로드 → 파서 (UvvisXlsxParser, IcpCsvParser 등)
    ↓
[명확한 매핑] 소스별 구분
    ├─ 수질분석센터 → 수질분석센터_결과
    ├─ 비용부담금 → 비용부담금_결과
    └─ 처리시설 → 처리시설_결과
    ↓
시험기록부 저장 (공용 *_시험기록부 테이블)
```

## 테이블 명명 규칙 (New Standard)

### 패턴: `{소스}_{목적}`

```
소스 분류:
├─ 수질분석센터     (수질 분석 서비스)
├─ 비용부담금       (폐수배출업소 분석)
├─ 처리시설         (처리시설 측정)
├─ 폐수배출업소     (기업 정보)
└─ ... (기타 도메인)

목적 분류:
├─ _마스터         (마스터 정보: 시설명, 시료명 등)
├─ _결과           (분석 결과: 최종값 저장)
├─ _시험기록부     (상세 기록: 측정값, 농도값 등)
├─ _의뢰           (의뢰 정보)
└─ ... (기타)
```

**일관성 예시**:
```
✓ 수질분석센터_결과
✓ 비용부담금_결과
✓ 처리시설_결과
✓ 생물학적_산소요구량_시험기록부
✓ 부유물질_시험기록부
✓ 처리시설_마스터
✓ 폐수배출업소_분석결과
```

**설계 이점**:
1. **일관성**: 모든 결과 테이블이 `{소스}_결과` 패턴

3. **유지보수**: 새로운 개발자도 명명규칙 이해 용이
4. **확장성**: 향후 추가 소스/목적 추가 시 일관성 유지
