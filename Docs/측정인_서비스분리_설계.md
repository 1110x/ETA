# 측정인 서비스 분리 설계

> 작성일: 2026-03-29  
> 목적: "측정인 전송" / "측정인 LOGIN" / "자료TO측정인" 세 기능의 역할·진입점·코드 경계를 명확히 구분

---

## 1. 서비스 분리 개요

| 기능명 | 위치(메뉴) | 목적 | 입력 | 출력 |
|--------|-----------|------|------|------|
| **측정인 전송** | 견적/의뢰서 → 헤더 버튼 | 의뢰계획 자동 작성 | 의뢰 트리뷰 선택 노드 | 측정인.kr 계획 작성 모달 자동 입력 + 저장 |
| **측정인 LOGIN** | 시험성적서 → BT7 | ETA DB 업데이트·마이그레이션 | 측정인.kr 로그인 세션 | 채취지점 DB / 분석항목 DB UPSERT |
| **자료TO측정인** | 시험성적서 → BT8 | 분석결과를 측정인.kr에 입력 | 선택 시료의 분석결과 | 측정인.kr 성적서 제출 폼 자동 입력 |

## 2. 현재 구현 현황

### 2-1. 측정인 전송 (`AnalysisRequestListPanel`)

- **진입**: `Views/Pages/AnalysisRequestListPanel.axaml.cs` — 헤더 `🌐 측정인 전송` 버튼
- **핸들러**: `BtnMeasurer_Click` (L643)
- **주요 로직**: `TryInjectRequestDataAsync` (L737)
  - CDP → 측정인.kr 탭 탐색
  - 부모노드(레코드)별 for 루프
  - Steps A~H: 계약→현장→환경기술인→시작일→계획번호→차량→분석항목→인력→목적 순 입력
  - 각 레코드 `insertFieldPlanBtn` 클릭 → 저장
- **사전 다이얼로그**: `MeasurerSendDialog` — 레코드별 목적(SELF/CF) + 인력 선택
- **관련 서비스**: `MeasurerService.FindContractBySamplingPoint()`, `AgentService`

**관여하지 않는 것**: DB 스크래핑/마이그레이션, 분석결과 입력

---

### 2-2. 측정인 LOGIN (`MeasurerLoginWindow`)

- **진입**: 시험성적서 → BT7 → `MainPage.BT7_Click` → `new MeasurerLoginWindow().Show(this)`
- **클래스**: `Views/MeasurerLoginWindow.axaml.cs`
- **주요 역할**:
  1. Edge 브라우저 원격 디버깅 포트(9222) 실행 및 측정인.kr 로그인
  2. 로그인 성공 후 페이지에 **"ETA 계약 DB 업데이트"** / **"ETA 분석DB 업데이트"** 버튼 자동 주입
  3. 각 버튼 클릭 시 채취지점·분석항목 데이터를 스크래핑 → `MeasurerService`에 UPSERT
- **관련 서비스**: `MeasurerService.SaveSamplingPoints()`, `MeasurerService.SaveAnalysisItems()`
- **부산물**: `LoginSucceeded` 프로퍼티 → 다른 기능(측정인 전송)에서 로그인 상태 재사용

> ⚠ 이 창은 **측정인 전송** 기능이 브라우저 탭을 못 찾을 때 보조적으로도 사용되므로,  
> `MeasurerLoginWindow` 클래스 자체는 공유 유지. 단, 진입점 목적을 명확히 구분.

---

### 2-3. 자료TO측정인 (`DataToMeasurerWindow`)

- **진입**: 시험성적서 → BT8 → `MainPage.BT8_Click` → `new DataToMeasurerWindow(_testReportPage?.SelectedSample).Show(this)`
- **클래스**: `Views/DataToMeasurerWindow.cs`
- **주요 역할**:
  1. 선택된 시료(`SampleRequest`)의 분석결과 로드
  2. `MeasurerInputRow` 모델로 변환 → DataGrid 표시 (항목·법적기준·분석결과·분석기기 등)
  3. 측정인.kr 현재 탭 제목 확인 (`CheckMeasurerPageAsync` → CDP `edit_title` h2 읽기)
  4. "측정인에 입력" 클릭 → `MeasurerLoginWindow` 열어 로그인 → 결과 입력 자동화
- **관련 서비스**: `AnalysisService`, `AnalysisRequestService`
- **CDP 범위**: `GetEditTitleFromCdpAsync` — 현재 탭 제목만 읽음 (분석결과 실제 입력은 미구현)

## 4. 데이터 흐름 요약

### 측정인 전송 (의뢰계획 작성)
```
의뢰 DB (의뢰_접수)
    ↓  AnalysisRequestListPanel._rowCache
MeasurerSendDialog (목적·인력 per-record)
    ↓  purposeValues[], empIdsPerRecord[][]
TryInjectRequestDataAsync
    ↓  CDP WebSocket JS 주입
측정인.kr 계획 작성 모달 → insertFieldPlanBtn
```

### 측정인 LOGIN (DB 업데이트)
```
측정인.kr 계약/현장 관리 페이지
    ↓  CDP 스크래핑 (ETA 계약 DB 업데이트 버튼)
MeasurerService.SaveSamplingPoints()
    → SQLite 측정인_채취지점 테이블 UPSERT

측정인.kr 분석항목 관리 페이지
    ↓  CDP 스크래핑 (ETA 분석DB 업데이트 버튼)
MeasurerService.SaveAnalysisItems()
    → SQLite 측정인_분석항목 테이블 UPSERT
```

### 자료TO측정인 (분석결과 입력)
```
시험성적서 선택 시료 (SampleRequest)
    ↓  AnalysisService.GetAllItems() + AnalysisRequestService
DataToMeasurerWindow DataGrid (MeasurerInputRow)
    ↓  사용자 확인/수정
측정인.kr 분석결과 입력 페이지 (CDP 주입, 구현 예정)
```

## 5. 공유 컴포넌트

| 컴포넌트 | 공유 범위 | 비고 |
|---------|----------|------|
| `MeasurerLoginWindow` | 측정인 전송(보조) + 측정인 LOGIN(주) + 자료TO측정인(보조) | `LoginSucceeded` 공유 |
| `MeasurerService` | 측정인 전송 + 측정인 LOGIN | FindContractBySamplingPoint, SaveSamplingPoints 등 |
| `AgentService` | 측정인 전송 | 인력 고유번호 매핑 |
| CDP 헬퍼 (`CdpEvalAsync`, `ExtractCdpValue`) | 측정인 전송 + 자료TO측정인 | 현재 각 클래스에 중복 — 공통 유틸로 추출 권장 |

## 6. 향후 개선 과제

| 우선순위 | 항목 | 설명 |
|---------|------|------|
| 🔴 High | `DataToMeasurerWindow` 자동 입력 구현 | `OnInputClick`에서 로그인만 하고 실제 입력 미구현 |
| 🟡 Mid  | CDP 헬퍼 공통 추출 | `CdpEvalAsync`/`ExtractCdpValue` Service 레이어로 이동 |
| 🟡 Mid  | 채취지점 선택 Step C (`add_emis_fac_no`) | 현재 `add_emis_fac_no` 미구현 (현장 선택 후 채취지점 목록 로딩 대기) |
| 🟢 Low  | `MeasurerSendDialog` 인력 초기 자동 선택 | DB 시료채취자와 매칭된 인력 pre-select |
| 🟢 Low  | 로그인 상태 persistent 캐싱 | 브라우저 재시작 없이 런타임 내 세션 유지 |
