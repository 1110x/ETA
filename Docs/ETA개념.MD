## 1. 시스템 개요

**ETA**는 리뉴어스 수질분석센터의 업무 전반을 관리하는 데스크톱 LIMS(실험실 정보관리 시스템)이다.

| 구분 | 내용 |
|------|------|
| 플랫폼 | .NET 8.0 + Avalonia UI (Windows / macOS) |
| 데이터베이스 | MariaDB (서버) 또는 SQLite (로컬) |
| 주요 기능 | 처리시설 측정결과 관리, 비용부담금 폐수 분석, 수질분석센터 의뢰/결과/성적서 |
| 출력 형식 | Excel(xlsx), PDF |
| 외부 연동 | 측정인.kr (CDP WebSocket 자동화) |

### 사업 배경

- **처리시설**: 여수국가산단(중흥, 월내, 4단계) 및 율촌산단(율촌, 세풍, 해룡)의 폐수 처리시설
- **비용부담금**: 처리시설 이용료 부과 대상 — 여수폐수(LG화학 외 다수), 율촌폐수(한맥 외 소수)
- **수질분석센터**: 리뉴어스 수질분석센터에서 고객사와 측정대행 표준계약을 체결하여 분석을 대행하고 수수료를 부과

## 2. 세 가지 업무 도메인

```
+--------------------------------------------------------------+
|                          ETA 시스템                           |
+------------------+-------------------+----------------------+
|   처리시설        |    비용부담금       |    수질분석센터        |
|                  |                   |                      |
| 여수/율촌산단     | 여수폐수/율촌폐수  | 외부 고객사 측정대행  |
| 처리시설 측정     | 비용부담 업체 분석  | 견적 -> 의뢰 -> 성적서|
| 요일별 측정계획   | 채수 -> 분석 -> 결과| 계약 기반 수수료 청구 |
+------------------+-------------------+----------------------+
```

### 2-1. 처리시설

- **대상**: 여수국가산단(중흥, 월내, 4단계), 율촌산단(율촌, 세풍, 해룡)
- **운영**: 요일별 분석계획이 ETA 서버와 연동되어 자동으로 해당 일의 분석항목/시설명이 생성됨
- **결과 저장**: 처리시설_측정결과 테이블에 최종 결과값, 항목별_시험기록부에 원자료(Raw Data)와 QC DATA 저장

### 2-2. 비용부담금

- **대상**: 처리시설을 이용하는 공장(업체)
- **채수**: 리뉴어스 여수사업소(중흥/율촌) 소속 별도 채수담당자가 샘플링
- **접수**: 채수된 시료가 수질분석센터로 접수되어 분석 진행
- **결과 저장**: 폐수의뢰및결과 테이블

### 2-3. 수질분석센터

- **대상**: 측정대행 표준계약을 체결한 외부 고객사
- **업무 흐름**: 견적 작성 → 분석의뢰서 → 시험성적서 발행 → 측정인.kr 등록
- **결과 저장**: 분석의뢰및결과 테이블

## 3. SN 체계 및 채수 구분

### SN(시료번호) 형식

| 구분 | 형식 | 예시 | 설명 |
|------|------|------|------|
| 여수폐수 | `MM-DD-NN` | `04-13-01` | 월-일-순서 |
| 율촌폐수 | `[율촌]MM-DD-NN` | `[율촌]04-13-01` | 접두사 포함 |
| 세풍폐수 | `[세풍]MM-DD-NN` | `[세풍]04-13-01` | 접두사 포함 |
| 수질분석센터 | 관리번호 방식 | `2026-여수-001` | 별도 관리번호 |

### 구분(소스구분) 코드

| 코드 | 내용 |
|------|------|
| 여수 | 여수국가산단 비용부담금 시료 |
| 율촌 | 율촌산단 비용부담금 시료 |
| 세풍 | 율촌산단 세풍지구 시료 |
| 처리시설 | 처리시설 자체 측정 시료 |

## 4. 도메인별 업무 흐름

### 4-1. 처리시설

```
측정계획 설정 (요일별 분석항목/시설명 체크)
    ↓ 서버 동기화
매일 자동으로 해당일 분석항목 목록 생성
    ↓
분석담당자 → 분석결과 입력 (처리시설 > 측정결과 메뉴)
    ↓
처리시설_측정결과: 결과값 업데이트
항목별_시험기록부: 원자료(Raw Data) + QC DATA 저장
```

### 4-2. 비용부담금

```
채수담당자 → 현장 채수
    ↓
수질분석센터 접수 (SN 생성: MM-DD-NN)
    ↓
폐수의뢰및결과 테이블에 의뢰 레코드 등록
    ↓
분석담당자 → 분석결과 입력 (비용부담금 > 분석결과입력 메뉴)
    ↓
엑셀 분석파일 첨부 → 파서 자동 인식 → SN 매칭
    ↓
폐수의뢰및결과: 결과값 반영
항목별_시험기록부: 원자료 저장
```

### 4-3. 수질분석센터

```
견적 작성 (QuotationService)
    ↓
분석의뢰서 생성 (AnalysisRequestService)
    ↓
현장 채수 → 시료 접수
    ↓
분석결과 입력 + 시험기록부 작성
    ↓
시험성적서 출력 (Excel/PDF)
    ↓
측정인.kr 자동 입력 (CDP 연동)
```

| SS | 부유물질 | 단일 | |
| T-N | 총질소 | 단일 | UV 분광광도계 |
| T-P | 총인 | 단일 | UV 분광광도계 |
| 페놀류 | Phenols | 단일 | UV 분광광도계 |
| 시안 | CN | 단일 | UV 분광광도계 |
| 6가크롬 | CR6 | 단일 | UV 분광광도계 |
| 색도 | COLOR | 단일 | UV 분광광도계 |
| ABS | ABS | 단일 | UV 분광광도계 |
| 불소 | FLUORIDE | 단일 | UV 분광광도계 |

| GCMS | 기체크로마토그래프-질량분석기 | 다성분 | VOC, 유기인 등 |
| ICP | 유도결합플라즈마 | 다성분 | 금속류 |

| PFAS | 과불화화합물 | 다성분 | PFOA, PFOS, PFBS |
| ECO | 생태독성 | 통계계산 | TSK/Probit, TU |
| 대장균 | 총대장균군 | 단일 | 처리시설 전용 |

### 5-2. 2계층 저장 구조

```
분석파일 (엑셀/CSV/PDF)
    ↓ 파서 실행
ExcelRow { SN, 시료명, D1, D2, Fxy, P, Result, CompoundName }
    ↓
+------------------------------------------+
| 1계층: 항목별_시험기록부 테이블            |  Raw Data (원자료 + QC)
|   {성분명}_시험기록부                     |
|   흡광도, ISTD, 농도, 기울기, 절편        |
|   ST1_Area, ST2_Area, ...               |
+------------------------------------------+
    ↓ 계산 결과만
+------------------------------------------+
| 2계층: 의뢰및결과 테이블                  |  최종 결과값
|   폐수의뢰및결과                          |
|   처리시설_측정결과                       |
|   분석의뢰및결과                          |
+------------------------------------------+
```

### 5-3. 검증 그리드 상태

파일 첨부 후 Show2에 검증 그리드가 표시된다:

| 상태 | 의미 | 토글 기본값 |
|------|------|------------|
| 초록 | 의뢰 존재 + 미입력 → 입력 가능 | ON |
| 노랑 | 의뢰 존재 + 이미 결과 있음 → 덮어쓰기 | ON |
| 빨강 | 의뢰에 해당 SN 없음 → 미매칭 | OFF (비활성) |

> **주의**: 빨간(미매칭) 행은 의뢰 테이블에 해당 SN이 없다는 뜻.  
> 입력 버튼을 눌러도 의뢰 결과 컬럼에는 반영되지 않는다.  
> 시험기록부(원자료)는 저장되지만, 최종 결과 집계에는 포함되지 않는다.  
> **진단**: 입력 후 `Logs/ImportDebug.log` 에서 SKIP(비활성) 메시지 확인.

### 5-4. 다성분 동시분석 (GC/ICP/PFAS)

```
GcInstrumentParser.Parse(파일)
    ↓
GcInstrumentFile.Compounds[N]
  각 성분별: Cal행 + 시료행 + 기울기/절편/R
    ↓ 성분별 분리
CompoundName ("Benzene") → _compoundToAnalyte → "벤젠"
    ↓
벤젠_시험기록부 테이블에 저장
폐수의뢰및결과.벤젠 컬럼에 결과값 반영 (UpdateDynamicValue)
```

### 5-5. 농도 계산 공식

분석항목별 농도 계산 공식은 `분석정보.농도계산공식` 컬럼에 텍스트로 저장된다.

**EvaluateFormula에 제공되는 변수:**

| 변수명 | 값 | 설명 |
|--------|-----|------|
| `흡광도` | D1 | UV: 흡광도 / GC: 응답값(Resp.) |
| `Area` | D1 | GC 공식 호환 alias |
| `Resp` | D1 | GC 공식 호환 alias |
| `ISTD` | D2 | GC ISTD 응답값 |
| `절편` | 검량선 절편 | |
| `기울기` | 검량선 기울기 | |
| `시료량` | 시료 체적 | 0이면 1.0으로 처리 |
| `희석배수` | 희석 배율 | |

**공식 예시:**
```
( 흡광도 - 절편 ) / 기울기 × 희석배수
( Area / ISTD - 절편 ) / 기울기 × 희석배수
```

## 8. 파서 시스템

### 8-1. 전체 흐름

```
파일 업로드
    ↓ 텍스트 추출
    +- SignatureClassifier.ClassifyTopK()  1순위: DB 규칙 기반
    +- AiParserClassifier.PredictTopK()   2순위: ONNX AI 보완
    ↓
후보 목록 다이얼로그 → 사용자 선택 → 파서 실행 → ExcelRow[]
```

### 8-2. 파서 생성기

새 장비 파일 형식에 맞는 파서를 자동 생성하는 도구.

**자동 감지**: 구분자, 헤더 행 수, 표준물질 키워드(STD/Cal/Blank), 데이터 열  
**생성 결과**: `*Row`, `*InstrumentFile`, `*Parser` C# 클래스 템플릿

### 8-3. 시그니처 분류기

```
DB: 파서_시그니처 테이블
  헤더_키워드      × 3 (가장 중요)
  표준물질_키워드  × 2
  시료명_키워드    × 1

점수 = 매칭 가중치 합 / 전체 가중치 합
임계값: 0.15 이상 → 분류 성공
```

### 8-4. GC 파서 상세

GC CSV 파일은 성분별 블록 구조:

```
[Benzene]
  검량선 행: STD1~STD6 (Resp, Conc)
  시료 행: SN, Resp, ISTD Resp, FinalConc
  기울기, 절편, R²
```

→ GcInstrumentFile.Compounds[] → 성분별 분리 → CompoundName 라우팅

## 9. 생태독성 통계분석

물벼룩 급성독성시험(ES 04704.1c)에서 LC50 계산, TU = 100 / LC50.

| 방법 | 알고리즘 |
|------|---------|
| TSK | Trimmed Spearman-Karber (Abbott 보정 + PAVA) |
| Probit | 가중최소제곱 반복회귀 |

**DB 저장**: `생태독성_시험기록부` 테이블, 처리시설_측정결과/폐수의뢰및결과의 TU 컬럼 반영

## 10. 측정인.kr 연동

```
[ETA 앱] --CDP WebSocket--> [Edge 브라우저] --> [측정인.kr]
  (C#)      port 9222                           (jQuery + select2)
```

**자동 입력 순서**: 근거계약 → 현장 → 환경기술인 → 분석시작일 → 분석항목 → 측정인력 → 측정목적

**핵심**: select2는 `option.selected + click() + jQuery trigger('select2:select') + trigger('change')` 조합 필수.

## 11. 화면 구성 및 레이아웃

```
+-------------------------------------------------------------------+
| [로고]  수질분석센터  비용부담금 관리  처리시설  관리자  [사용자명]   |
+-------------------------------------------------------------------+
| [BT1]  [BT2]  [BT3]  [BT4]  [BT5]  [BT6]  [BT7]              |
+--------------------+----------------------------------------------+
|                    |                                              |
|       Show1        |                Show2                        |
|   트리뷰 / 메뉴    |           목록 / 그리드 / 폼                |
|                    +----------------------------------------------+
|                    |                                              |
|                    |                Show3                        |
|                    |           편집 폼 / 액션 패널               |
+--------------------+----------------------------------------------+
```

### 분석결과 입력 화면

| 패널 | 내용 |
|------|------|
| Show1 | 카테고리 버튼 18개 + 날짜 트리뷰 |
| Show2 | 검증 그리드 (상태 + 토글) |
| Show3 | 선택 시료 항목별 입력 폼 |
| Show4 | 항목별 입력 현황 프로그레스 바 |

### 서브메뉴 버튼

| 버튼 | 기능 |
|------|------|
| BT1 새로고침 | 트리뷰 + 그리드 재로드 |
| BT2 검증 | 파일 재첨부 + 검증 |
| BT3 입력 | 토글 ON인 행만 DB 반영 |
| BT4 출력 | 그리드 클립보드 복사 |
