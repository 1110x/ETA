# 파서 시그니처 분류기 아키텍처

## 배경 및 목표

분석장비가 변경되거나 추가될 때마다 C# 파서를 직접 작성해야 하는 문제를 해결하기 위해 도입.  
파서 생성기에서 파일 구조를 자동 분석 → 시그니처 DB에 저장 → 새 파일 자동 분류까지 이어지는 파이프라인.

---

## 전체 흐름

```
[파서 생성기] 분석 파일 업로드
    ↓ 텍스트 분석 (규칙 기반)
    ↓ 표준물질 키워드, 헤더 키워드, 시료명 패턴 추출
    ↓ C# 파서 코드 템플릿 생성
    ↓ [시그니처 저장] 버튼
    → DB: 파서_시그니처 테이블

[새 파일 첨부 시] (WasteAnalysisInputPage 등)
    ↓ 텍스트 추출
    ↓ SignatureClassifier.ClassifyTopK()  ← 1차: 시그니처 (규칙 기반)
    ↓ AiParserClassifier.PredictTopK()   ← 2차: ONNX (확률 기반, 보완)
    → 통합 후보 목록 표시 → 사용자 선택
```

---

## 핵심 컴포넌트

### 1. 파서 생성기 (`ParserGeneratorPage`)

파일을 업로드하면 다음을 자동 분석:

| 항목 | 방법 |
|------|------|
| 구분자 | `,` / `\t` / `;` 빈도 비교 |
| 헤더 행 수 | 첫 행부터 수치 비율 50% 초과 시점까지 |
| 표준물질 키워드 | 첫 열에서 STD/Cal/Blank/BLK/QC/표준/검량 등 접두사 검출 |
| 시료명 패턴 | 표준물질 제외 행의 첫 열 패턴 분석 |
| 데이터 열 | 수치 비율 ≥40% 인 열 목록 |

분석 결과를 기반으로 C# 파서 클래스 템플릿 자동 생성 (`*Row`, `*InstrumentFile`, `*Parser`).

### 2. 시그니처 분류기 (`SignatureClassifier`)

**DB 테이블**: `파서_시그니처`

| 컬럼 | 설명 |
|------|------|
| 파서키 | "BOD", "GC", "TOC_Shimadzu" 등 |
| 표준물질_키워드 | 가중치 ×2 |
| 헤더_키워드 | 가중치 ×3 (가장 중요) |
| 시료명_키워드 | 가중치 ×1 |

**점수 계산**:
```
score = (매칭된 키워드 가중치 합) / (전체 가중치 합)
임계값: 0.15 이상이면 분류 성공
```

### 3. ONNX 분류기 (`AiParserClassifier`) — 보완 역할

AI 문서분류 학습 탭에서 Python으로 TF-IDF + LogisticRegression 훈련 →  
`Data/분석항목_분류.onnx` 생성 → C#에서 로컬 추론.

시그니처 분류기가 낮은 신뢰도를 보일 때 보완 역할 수행.  
**AI 학습 기능은 제거하지 않음 — 두 시스템이 공존함.**

---

## 분류기 우선순위

```
1순위: SignatureClassifier (시그니처 DB)
        → 결정론적, 빠름, 새 파서 추가에 즉시 반응
2순위: AiParserClassifier (ONNX)
        → 확률적, 학습 데이터 기반, 패턴 불명확한 파일에 유리
```

결과 표시 시 출처 배지: `시그니처` / `AI`

---

## 신규 장비 추가 워크플로

```
1. 장비에서 파일 출력 (CSV/TXT/Excel 등)
2. ETA → 파서 생성기 → 파일 업로드
3. 자동 분석 결과 확인 (표준물질·시료명·데이터 열)
4. 파서 타입 선택 → [시그니처 저장]
5. 이후 해당 장비 파일 첨부 시 자동 분류됨
6. (선택) AI 학습 탭에서 추가 학습 → ONNX 재생성으로 정확도 보완
```
