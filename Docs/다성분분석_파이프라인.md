# 다성분 동시분석 파이프라인 설계

## 배경

ETA의 분석결과입력 시스템은 **18개 분석 카테고리**를 지원한다. 이 중 일부는 "하나의 파일에서 여러 화합물/원소 결과가 동시에 나오는" **다성분 동시분석** 항목이다.

| 분류 | 카테고리 | 파일 예시 | 성분수 | 파서 포맷 |
|---|---|---|---|---|
| GC 다성분 | VOC10종 | VOC10.csv | 10 (1,1-DCE ~ Xylene) | VocMulti (ISTD) |
| GC 다성분 | VOC3종 | VOC3.csv | 3 (Vinylchloride, Acrylonitrile, Bromoform) | VocMulti (ISTD) |
| GC 다성분 | VOC2종 | (추후) | 2 | VocMulti (ISTD) |
| GC 단일 | 각종 | 나프탈렌.csv 등 15종 | 1 | VocSingle / SingleNoIstd / SingleExpConc |
| 유기인 | GCMS | 유기인.csv | 5개 묶음 (Org-P) | SingleExpConc |
| 금속류 | ICP | (PDF) | 다수 (Cu, Zn, Pb 등) | IcpPdfParser |
| 과불화 | PFAS | (PDF) | 3 (PFOA, PFOS, PFBS) | LcmsPfasPdfParser |

핵심 문제: 단일 항목(BOD, SS 등)과 달리, 다성분은 **하나의 파일에서 N개의 분석항목 결과**가 나온다. 이를 성분별로 분리하여 각각의 시험기록부 테이블에 저장하고, 의뢰및결과 테이블의 해당 컬럼에 반영해야 한다.

## 핵심 설계: CompoundName 기반 라우팅

### ExcelRow.CompoundName 필드

모든 분석 데이터는 `ExcelRow` 객체로 통일된다. 다성분 항목을 위해 `CompoundName` 필드를 추가:

```
ExcelRow
  ├── 시료명: "Benzene | RE-01.D"   (UI 표시용 합성명)
  ├── CompoundName: "Benzene"        (라우팅 키)
  ├── SN: "RE-01"                    (시료 식별)
  ├── D1: "11841"                    (Resp. 응답값)
  ├── D2: "2529"                     (ISTD Resp.)
  ├── Fxy: "0.008"                   (기기 계산 농도)
  ├── Result: "0.008"                (최종 결과)
  ├── P: "1"                         (희석배수)
  └── IsControl: false               (정도관리 시료 여부)
```

단일 항목(BOD, SS 등)에서는 `CompoundName = ""` — 기존 로직에 영향 없음.

### 영문 성분명 → 한글 분석항목명 매핑

GC CSV 파일은 영문 성분명("Benzene")을 사용하지만 DB는 한글("벤젠")을 사용한다:

```csharp
_compoundToAnalyte = {
    "Benzene" → "벤젠", "Toluene" → "톨루엔", "Xylene" → "자일렌",
    "Cu" → "구리", "Zn" → "아연", "Pb" → "납",
    "PFOA" → "PFOA", ...
}
```

### 다성분 카테고리 판별

```csharp
IsMultiCompoundCategory("GCMS") → true   // GC/MS
IsMultiCompoundCategory("ICP")  → true   // 금속류
IsMultiCompoundCategory("PFAS") → true   // 과불화화합물
IsMultiCompoundCategory("IC")   → true   // 이온크로마토
IsMultiCompoundCategory("BOD")  → false  // 단일 항목
```

## 데이터 흐름

### 1. 파싱 단계

```
GC CSV 파일 (VOC10.csv)
    ↓
GcInstrumentParser.Parse()
    ↓
GcInstrumentFile
  ├── Format: VocMulti
  ├── AnalysisTime: "2026-03-06"
  └── Compounds[10]:
      ├── [0] 1,1-DCE: Cal[6] + Rows[17] + Slope/Intercept/R
      ├── [1] DCM:     Cal[6] + Rows[17] + Slope/Intercept/R
      ├── ...
      └── [9] Xylene:  Cal[6] + Rows[17] + Slope/Intercept/R
```

### 2. 변환 단계 (ParseGcInstrumentFile)

```
GcInstrumentFile
    ↓ (flatten: 성분 × 시료 → ExcelRow 목록)
ExcelRow[170]   (10성분 × 17시료, Cal 제외, 정도관리 IsControl=true)
    ↓
_categoryExcelData["GCMS"] = ExcelRow[170]
_activeItems = ["1,1-DCE", "DCM", ..., "Xylene"]  (동적 구성)
_categoryDocInfo["GCMS"].GcCompoundCals = GcCompoundCalInfo[10]
```

**핵심**: `_activeItems`가 파싱된 성분명 배열로 동적 구성됨. 기존에는 `Array.Empty<string>()`이라 SaveRawData가 실행 안 됨.

### 3. UI 표시 (Show2 그리드)

```
┌──────────────────────────────────────────────────────────┐
│ GC [VocMulti] — 10성분                                    │
├──────────────────────────────────────────────────────────┤
│ 검량선 테이블 (성분별 3행씩):                              │
│   1,1-DCE  | ST-1~6 농도 | 기울기 | 절편 | R²            │
│            | ST-1~6 Resp. |                               │
│            | ST-1~6 ISTD  |                               │
│   DCM      | ...                                          │
│   ...                                                     │
├──────────────────────────────────────────────────────────┤
│ 시료 데이터 그리드:                                        │
│ [V] [입력] SN      시료명              Resp. ISTD  농도  │
│ [V] [ON]  RE-01   1,1-DCE | RE-01.D    0    2529   ND   │
│ [V] [ON]  RE-01   DCM | RE-01.D       17110 2529  0.000 │
│ [V] [ON]  RE-01   Chloroform | RE-01.D 10840 2529 0.012 │
│ ...                                                       │
│ [ ] [OFF] BK-     1,1-DCE | BK-.D      -    2907   ND   │ ← IsControl
└──────────────────────────────────────────────────────────┘
```

### 4. 저장 단계 (ImportData → SaveRawData)

```
각 ExcelRow에 대해:

  row.CompoundName = "Benzene"
      ↓
  SaveRawData case "GCMS":
      tableName = "Benzene_시험기록부"   ← SafeName(compound) + "_시험기록부"
      compoundCal = GcCompoundCals.Find("Benzene")  ← 해당 성분 검량선
      ↓
  UpsertGcData("Benzene_시험기록부", ...)
      INSERT/UPDATE: 분석일, SN, 시료명, 농도, ISTD,
                     ST1-7 (농도/값/ISTD), 기울기, 절편, R값
```

VOC10 → 10개 테이블 생성:
`1_1DCE_시험기록부`, `DCM_시험기록부`, `Chloroform_시험기록부`, ...

### 5. 의뢰및결과 연동

3가지 소스 타입별로 다성분 분기:

```
                     ┌─ 폐수배출업소 ─→ UpdateDynamicValue("폐수의뢰및결과", id, "벤젠", "0.008")
CompoundName         │                    ALTER TABLE ADD COLUMN IF NOT EXISTS
"Benzene"            │
  → ResolveAnalyteName │─ 수질분석센터 ─→ UpdateResultValue(ar.Id, "벤젠", "0.008")
  → "벤젠"            │                    (이미 동적 컬럼 지원)
                     │
                     └─ 처리시설     ─→ target["벤젠"] = "0.008"
                                         (Dictionary 기반 동적 값)
```

### 검량선 계산

각 성분별 최소자승법 선형회귀:
- y = ax + b (x = 농도, y = Resp / ISTD Resp)
- 기울기(a), 절편(b), 상관계수(R) 계산
- ST-1 ~ ST-7 까지 최대 7점 지원

## DB 테이블 구조

### 시험기록부 (성분별 1테이블)

VOC 스키마 (GCMS 기본):
```sql
CREATE TABLE `Benzene_시험기록부` (
    분석일 TEXT, SN TEXT, 업체명 TEXT, 구분 TEXT, 시료명 TEXT,
    농도 TEXT, ISTD TEXT,
    ST1_농도 TEXT, ST1_값 TEXT, ST1_ISTD TEXT,
    ST2_농도 TEXT, ST2_값 TEXT, ST2_ISTD TEXT,
    ...  (ST3 ~ ST7 동일)
    기울기 TEXT, 절편 TEXT, R값 TEXT,
    비고 TEXT, 등록일시 TEXT
);
```

UNIQUE KEY: `(분석일, SN, 시료명)` — 같은 날짜·시료의 덮어쓰기 허용.

### 의뢰및결과 (동적 컬럼)

폐수의뢰및결과 테이블은 7개 고정 컬럼(BOD, TOC, SS, T-N, T-P, N-Hexan, Phenols) + 5개 UV 컬럼(시안, 6가크롬, 색도, ABS, 불소)을 가지며, 다성분 항목은 `UpdateDynamicValue`로 필요시 컬럼을 자동 추가한다.

```sql
ALTER TABLE `폐수의뢰및결과` ADD COLUMN `벤젠` TEXT DEFAULT '';
UPDATE `폐수의뢰및결과` SET `벤젠` = '0.008' WHERE Id = 123;
```

## SafeName 변환 규칙

성분명 → 테이블명 변환 시 특수문자 처리:

```
"1,1-DCE"     → "1_1DCE"        (콤마→언더스코어, 하이픈→제거)
"T-N"         → "TN"            (하이픈→제거)
"Carbon Chloride" → "Carbon_Chloride" (공백→언더스코어)
"N-Hexan"     → "NHexan"
```

## 정도관리 시료 처리

GC 파일에는 분석의 정확성을 검증하는 정도관리 시료가 포함된다:

| Type | 의미 | 시험기록부 | 의뢰및결과 |
|---|---|---|---|
| Cal | 검정곡선 표준용액 | 제외 (검량선 데이터로 별도 저장) | 제외 |
| Blank / BK | 바탕시료 | **포함** (IsControl=true) | 제외 |
| FBK | 필드바탕시료 | **포함** | 제외 |
| MBK | 방법바탕시료 | **포함** | 제외 |
| QC / CCV | 검정확인 | **포함** | 제외 |

기록부는 분석이 정상적으로 수행되었다는 증거이므로, 정도관리 시료도 반드시 시험기록부에 보존한다.

## 신규 다성분 항목 추가 가이드

새로운 다성분 항목(예: VOC2)을 추가할 때:

1. **파서가 기존 GcInstrumentParser를 사용하면** — 추가 코드 불필요. CSV 파일 첨부만으로 자동 인식.

2. **`_compoundToAnalyte` 매핑에 성분 추가** — 영문명 → 한글 DB 컬럼명:
   ```csharp
   ["NewCompound"] = "새화합물",
   ```

3. **DB 테이블 자동 생성** — `WaterCenterDbMigration`이 `분석정보` 테이블 기반으로 시험기록부 테이블을 자동 생성. 신규 성분이 `분석정보`에 등록되어 있어야 함.

4. **의뢰및결과 컬럼** — `UpdateDynamicValue`가 없으면 자동 `ALTER TABLE ADD COLUMN`.
